开始前先看（1分钟）

  1. 先确认你遇到的是“访问/显示问题”（比如页面打不开、图片不出、样式乱、下载失败）。如果是账号/权限/套餐问题，这个工具不解决。
  2. 修复类功能需要管理员权限：
    • Windows 右键程序 → 以管理员身份运行
    • macOS 若提示权限不足，需要用管理员方式运行（或联系技术同事协助）
  3. 修复后一定要刷新：优先“刷新网页”，不行就“完全退出浏览器再打开”，再不行“重启电脑”。

点进“问题修复”弹窗后，怎么操作？

弹窗里你只需要关注这几个按钮/区域（名字跟软件里一致）。

  1. 问题描述 / 解决方案：帮助你确认“选对问题了”。
  2. 立即修复：真正开始修复（会修改 hosts + 清 DNS）。
  3. 预览修改：当前版本会提示“后续实现”，可以先忽略。
  4. 修复进度条：显示正在做什么（获取 IP、备份 hosts、修改 hosts、清 DNS）。
  5. Safari 特例：Safari 问题弹窗不会“自动修复”，而是给你一步步的清缓存引导，按钮是 我已了解。

“问题示例图”怎么用？（对照你屏幕上的现象）

软件弹窗里会显示示例截图。你也可以在这里先看一眼，快速确认自己遇到的是哪一类问题。

不点“一键修复”，你也可以手动处理（适合：没有管理员权限/软件打不开/需要按技术同事指示操作）

一键修复的本质是：改 hosts（让电脑访问指定域名时走指定 IP）+ 清 DNS 缓存（让修改立刻生效）。

  1. 先确定你要处理的是哪一类问题（下面有“域名对照表”）。
  2. 再按你的系统（Windows/macOS）去改 hosts。
  3. 最后清 DNS + 刷新浏览器。

域名对照表（你要往 hosts 里加什么）

你只需要复制对应问题的“域名行”。其中有些问题的 IP 是固定的；有些 IP 会变化（可按下面步骤自己获取，无需技术同事）。

【IP 会变化时，先这样拿到“最优IP”（无需技术同事）】 方法A（推荐）：打开 17ce.com → 选择 PING → 输入域名 → 取“最快/延迟最低”的节点 IPv4 方法B（你电脑有 Python 时）：在项目目录执行：python hosts/get_domain_ip.py <域名> 【预览图问题】（IP 会变化） <上一步获取的IP> preview.qiantucdn.com 【样式问题】（IP 会变化） <上一步获取的IP> js.qiantucdn.com 【主站样式】（IP 会变化） <上一步获取的IP> icon.qiantucdn.com 【下载页面无法访问】（固定 IP） 47.104.5.133 dl.58pic.com 【云设计首页无法访问】（固定 IP） 118.190.104.146 y.58pic.com 【主站打不开】（固定 IP） 47.104.159.75 www.58pic.com 47.104.159.75 qiye.58pic.com 【下载失败/中断】（IP 可能变化，且可能是多个域名） <对应域名获取的IP> proxy-rar.58pic.com <对应域名获取的IP> proxy-vip.58pic.com <对应域名获取的IP> proxy-vd.58pic.com 【卡片有标签但无内容】（这是“解绑”，不是绑定） 从 hosts 里删除含有 preview.qiantucdn.com 的那一行

说明：预览图/样式/下载代理这些域名的最优 IP 可能会变，所以教程里不写死 IP，避免误导。你可以让技术同事把“IP + 域名”整行发你，直接复制进 hosts。

Windows：如何找到并修改 hosts（图文版）

  1. 打开 hosts 文件路径：
    C:\Windows\System32\drivers\etc\hosts
  2. 用管理员权限打开记事本：
    • 开始菜单搜索：记事本 → 右键 → 以管理员身份运行
  3. 在记事本里打开 hosts：
    • 文件 → 打开 → 粘贴路径
    • 右下角文件类型从“文本文档 (*.txt)”切换到 所有文件 (*.*)
    • 选择 hosts 打开
  4. 先备份：把文件另存一份，比如 hosts.backup（同目录或桌面都行）。
  5. 在文件末尾追加：把“域名对照表”里的行复制粘贴进去（每行一个域名）。
  6. 保存：Ctrl+S。
  7. 清 DNS 缓存（让修改立刻生效）：
    以管理员身份打开 CMD，然后执行：
    ipconfig /flushdns
  8. 验证：刷新网页（Ctrl+F5），不行就完全退出浏览器重开。

macOS：如何找到并修改 hosts（命令版）

macOS 的 hosts 路径固定在：

/etc/hosts
  1. 备份 hosts（推荐先做）：
    sudo cp /etc/hosts /etc/hosts.backup
  2. 用管理员权限编辑（选择一种你熟悉的编辑器）：
    sudo nano /etc/hosts
    如果你不熟悉 nano：把要添加的行复制给技术同事，让他们远程协助更稳。
  3. 在文件末尾追加：把“域名对照表”里的行粘贴进去（每行一个域名）。
  4. 保存并退出（nano）：
    按 Ctrl+O 保存 → 回车确认 → Ctrl+X 退出
  5. 清 DNS 缓存：
    sudo dscacheutil -flushcache
    sudo killall -HUP mDNSResponder
  6. 验证：刷新网页（⌘+R），不行就完全退出浏览器重开。

如果你卡在“没有权限”，该怎么做？（给非技术同事的处理方式）

  1. Windows：请找 IT/管理员账号同事帮你用“以管理员身份运行”打开记事本再改。
  2. macOS：需要输入管理员密码才能改 /etc/hosts；如果你没有密码，请把“域名对照表里你要加的那几行”发给有权限的同事代操作。
  3. 替代方案：如果你能打开软件但没权限修复，可以先用工具箱做“检查 Hosts 配置/收集系统信息”，把信息发技术同事。

最简单判断法（优先用）

打开工具主界面，找到最接近你“现象”的卡片，然后点 一键修复。

  1. 只要预览图不显示/加载慢 → 选「预览图问题」
  2. 只要页面布局乱 / CSS 像没加载 → 选「样式问题」或「主站样式」
  3. 只要下载页面提示无法访问 → 选「下载问题」
  4. 只要云设计首页无法访问 → 选「云设计问题」
  5. 只要卡片有文字标签但没有内容图 → 选「卡片加载异常」
  6. 只要主站/企业版直接打不开 → 选「主站打不开」
  7. 只要下载到一半断/网络错误 → 选「下载失败」
  8. 只要macOS 用 Safari 打不开/异常 → 选「Safari无法打开」

不知道选哪个？按这条“排除顺序”来

适合你只知道“就是不对劲”，但说不清具体哪种问题。

  1. 先看是不是主站打不开：如果 `www.58pic.com` / `qiye.58pic.com` 进不去 → 直接选「主站打不开」
  2. 再看是不是下载相关：下载页打不开 →「下载问题」；下载中断/网络错误 →「下载失败」
  3. 再看是图片/样式：预览图问题 →「预览图问题」；样式乱/图标缺失 →「样式问题/主站样式」
  4. macOS Safari 特例：Safari 打不开或显示怪 →「Safari无法打开」按步骤清缓存

🖼️ 预览图问题 一键修复

现象：主站卡片预览图不显示、加载很慢。

  1. 判断：页面能打开，但卡片图像加载失败或极慢。
  2. 处理方法：点「预览图问题」→「立即修复」。
  3. 修复原理：绑定 preview.qiantucdn.com 到更合适的 IP + 清理 DNS。

🎨 样式问题（页面样式异常） 一键修复

现象：下载页样式乱、布局错位、像“没加载样式”。

  1. 判断：文字能看到，但排版/按钮样式明显不对。
  2. 处理方法：点「样式问题」→「立即修复」。
  3. 修复原理：绑定 js.qiantucdn.com + 清理 DNS。

🎨 主站样式（样式丢失） 一键修复

现象：主站样式/图标资源加载不出，页面看起来“裸奔”。

  1. 判断：主站能打开，但图标/样式缺失。
  2. 处理方法：点「主站样式」→「立即修复」。
  3. 修复原理：绑定 icon.qiantucdn.com + 清理 DNS。

⬇️ 下载问题（无法访问） 一键修复

现象：点击下载素材后，下载页提示“无法访问网站”。

  1. 判断：不是下载中断，而是“下载页面”本身打不开。
  2. 处理方法：点「下载问题」→「立即修复」。
  3. 修复原理：绑定 dl.58pic.com 到固定 IP（工具内置）+ 清理 DNS。

☁️ 云设计问题（首页无法访问） 一键修复

现象：云设计首页打不开，提示无法访问。

  1. 判断：只在云设计业务入口出问题。
  2. 处理方法：点「云设计问题」→「立即修复」。
  3. 修复原理：绑定 y.58pic.com 到固定 IP（工具内置）+ 清理 DNS。

📦 卡片加载异常（有标签但无内容） 一键修复

现象：首页卡片区域只显示标签文字，但卡片内容/图片不出来。

  1. 判断：这种情况通常是之前绑定的规则反而导致异常。
  2. 处理方法：点「卡片加载异常」→「立即修复」。
  3. 修复原理：解除（解绑）preview.qiantucdn.com + 清理 DNS。

🚨 主站打不开 关键问题

现象：`www.58pic.com` 或 `qiye.58pic.com` 直接打不开，浏览器提示无法访问。

  1. 判断：不是“图片/样式缺失”，而是整站进不去。
  2. 处理方法：点「主站打不开」→「立即修复」。
  3. 修复原理：绑定主站域名到可用 IP + 清理 DNS。

⚠️ 下载失败（网络错误/中断） 常见

现象：下载开始了，但中途断开 / 提示网络错误。

  1. 判断：下载页能打开，但下载过程不稳定。
  2. 处理方法：点「下载失败」→「立即修复」。
  3. 修复原理：绑定下载代理域名（如 proxy-rar.58pic.com 等）+ 清理 DNS。

🧭 Safari无法打开（macOS） 按教程操作

现象：macOS 用 Safari 打不开页面或显示异常。

  1. 判断：换 Chrome 正常，但 Safari 不正常。
  2. 处理方法：点「Safari无法打开」，按弹窗步骤清除 58pic.com 网站数据（缓存/Cookie），然后重启 Safari。
  3. 注意：清除后可能需要重新登录。

工具箱：什么时候用

如果你“点了一键修复”但仍感觉不稳，或者需要把信息发给技术同事，优先用这里。

  1. 检查Hosts配置：想确认“是否真的修复成功/绑上了”就用它。
  2. 清除浏览器缓存：样式、图片更新不出来时很有用。清除前请先关闭浏览器。
  3. 清除DNS缓存：修复后不生效、或换网络后仍指向旧地址时使用。
  4. 检查浏览器版本：浏览器太旧可能导致兼容问题。
  5. 诊断下载问题：会提示常见原因（第三方下载工具、浏览器设置等）。
  6. 一键获取系统信息：需要把环境信息发给技术支持时用（比如系统版本、权限状态等）。

修复后如何验证（按顺序）

  1. 刷新页面（Windows: Ctrl+F5 / macOS: ⌘+R）。
  2. 完全退出浏览器再打开（不是只关掉标签页）。
  3. 清除DNS缓存（工具箱里一键清除）。
  4. 换浏览器：Safari 异常优先换 Chrome/Edge 验证是否为 Safari 单点问题。
  5. 仍不行：把“你选了哪个问题 + 修复结果提示 + 当前系统信息”发给技术同事。

失败/不生效怎么办（排障）

提示“需要管理员权限 / 权限不足”怎么办？

这是正常的：因为修复会改系统 hosts。

  • Windows：右键程序 → 以管理员身份运行。
  • macOS：如果你没有管理员权限，请把此问题交给有管理员权限的同事或技术支持。
点了“一键修复”，弹窗显示成功，但页面还是不对？
  • 先用工具箱 清除DNS缓存。
  • 再 清除浏览器缓存（清除前关闭浏览器）。
  • 再完全退出浏览器重开。
  • 还不行就重启电脑（让 DNS/网络栈彻底刷新）。
“卡片有标签但没内容”修复后更糟了？

这类问题通常是“绑定反而造成异常”，所以修复动作是解绑。

  • 先点「卡片加载异常」执行修复。
  • 修复后按“修复后验证”的顺序刷新/清 DNS。
  • 如果仍异常，尝试「预览图问题」或「样式问题」作为补充修复。
下载失败时，除了“一键修复”，我还要检查什么？
  • 尽量使用浏览器自带下载，不要用第三方下载器（如迅雷）。
  • 检查浏览器是否开启了某些“加速下载/云加速”类功能（可能影响稳定性）。
  • 换网络（手机热点）快速排除公司网络限制。
我需要把哪些信息发给技术同事？

建议复制以下内容发过去（越短越好，但要关键）：

1) 你遇到的现象（一句话）
2) 你点了哪个问题卡片（例如：预览图问题 / 下载问题）
3) 工具弹窗结果（成功/失败 + 报错文字）
4) 你做过的验证动作（刷新/清DNS/清缓存/换浏览器）
5) 你使用的系统和浏览器（Windows/macOS + Chrome/Safari）
